How our calculator program demonstrates the principles of object oriented programming

Throughout the creation of our calculator program, the fundamentals of object oriented programming were implemented. The program utilized encapsulation to bundle specific methods together for both testing purposes and for operational purposes. The calculator as a whole is the combination of encapsulated files and methods all working together. The separate files used to build the program structure utilized inheritance, which allowed for us to create classes that were built upon the existing classes. We did this in order to maintain the same behaviors within the new classes and to ensure that all the data being passed throughout the program is undergoing the same operational structure for correct results.


Within this program, we also were able to incorporate polymorphism, which is the ability of an object to behave in multiple form. In our case, we were able to create multiple instances of the calculator class which we could use to perform the different operations as well as test those operations without the need for re-defining a new class every time it was needed. This ability for reuse saves an incredible amount of development time and keeps the structure of the code clean with little room for redundancy errors.


The last principle of object oriented programming that was demonstrated in this project is abstraction. This can be explained simply by hiding implementation details and only showing the functionality to the user. When we understand what abstraction is, we can see that we are only focused on what the object does instead of how it functions. Here we utilized abstraction within the calculator class itself, where we extended the original class to create these new instances used for testing and carrying out operations. When an entity calls a method or another class, it has no insight into the functionality of this tool, just that it will properly execute the desired purpose it was called to do.
